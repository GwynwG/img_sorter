# JG-AI 模拟损伤光斑图分级识别方案

## Summary
基于 `D:\project\img_sorter\模拟图` 现有数据，做一个只面向“损伤光斑图”的本地识别基线系统。  
第一版交付形态是本地 `Gradio` 网页界面：上传单张损伤光斑图，输出：

- 级联位置：`第一级` / `第二级`
- 损伤程度：`4 档`
- 每项预测的置信度
- 结果说明：该程度是按模拟序列编号范围定义的粗略区间，不追求精确到 `img_0xx`

本期只使用模拟图训练和验证，不改现有业务代码，单独规划新的训练与推理流程。

## Key Changes
### 1. 数据定义与标签规则
使用现有目录作为唯一真值来源：

- `第一级/损伤光斑图` 与 `第二级/损伤光斑图` 作为模型输入图像
- `第一级` / `第二级` 目录名作为“级联位置”标签
- 文件名编号 `img_001` 到 `img_121` 作为损伤演化顺序
- 程度按等宽 `4` 档定义：
  - `S1`: `img_001` - `img_030`
  - `S2`: `img_031` - `img_060`
  - `S3`: `img_061` - `img_090`
  - `S4`: `img_091` - `img_121`

`损伤图` 不作为推理输入，也不进入第一版训练主链路；它只用于数据核对和后续物理解释参考。

### 2. 模型路线
继续使用 `Ultralytics`，但采用 `classify` 任务，不走 `detect`。

采用“两阶段”方案，落成 3 个分类模型：

- 模型 A：级联位置分类器
  - 输入：任意一张损伤光斑图
  - 输出：`第一级` / `第二级`
- 模型 B：第一级程度分类器
  - 输入：被模型 A 判为 `第一级` 的损伤光斑图
  - 输出：`S1` / `S2` / `S3` / `S4`
- 模型 C：第二级程度分类器
  - 输入：被模型 A 判为 `第二级` 的损伤光斑图
  - 输出：`S1` / `S2` / `S3` / `S4`

这样做的原因和约束已经固定：

- 先判级联位置，再判程度，更符合你现在的业务目标
- 两级联的光斑分布可能不同，程度模型分开训练更稳、更容易解释
- 程度只要求粗略范围，4 档分类比精确回归更合适

### 3. 统一预处理
训练和推理都使用完全一致的预处理：

- 读取原始 `2000x2000` 灰度图
- 固定裁切中心区域，去掉大面积无信息黑边
- 转成 3 通道输入 `Ultralytics classify`
- 统一缩放到轻量分类模型输入尺寸
- 保留轻量增强，避免破坏中心环纹结构

默认使用轻量分类预训练权重作为起点，例如 `yolo11n-cls` 级别的 backbone，不从零训练。

### 4. 数据切分与防泄漏规则
数据按编号“区段切分”，不做纯随机切分，避免相邻时间点同时落入训练和验证。

固定切分规则如下，两个级联都使用同一规则：

- 训练：
  - `001-024`
  - `031-054`
  - `061-084`
  - `091-114`
- 验证：
  - `025-030`
  - `055-060`
  - `085-090`
  - `115-121`

由此派生 3 套训练数据：

- 位置分类数据集：汇总两级联的训练/验证图像
- 第一级程度数据集：只含第一级训练/验证图像，并按 `S1-S4` 建类
- 第二级程度数据集：只含第二级训练/验证图像，并按 `S1-S4` 建类

### 5. 推理与界面
第一版对外接口固定为本地 `Gradio` 页面：

- 上传单张 `损伤光斑图`
- 页面显示原图
- 后端先跑模型 A，再根据结果路由到模型 B 或模型 C
- 页面输出：
  - 级联位置
  - 程度档位
  - 两个置信度
  - 对应编号区间解释，例如“中高损伤，对应模拟序列约 `91-121` 段”

第一版不要求批量 CSV，不要求桌面安装包，不要求损伤图输入。

## Public Interfaces
第一版固定一个本地交互入口：

- `Gradio` 单图上传页
- 输入：单张损伤光斑图文件
- 输出字段：
  - `cascade_level`: `第一级` / `第二级`
  - `severity_bin`: `S1` / `S2` / `S3` / `S4`
  - `cascade_confidence`
  - `severity_confidence`
  - `severity_range_text`

内部训练接口固定为 3 条独立分类训练流程，不与现有 `app.py` 复用。

## Test Plan
### 数据与标签验证
- 检查两级联下 `损伤光斑图` 是否各有 `121` 张
- 检查文件编号是否连续且一一对应
- 检查 4 档标签映射是否正确
- 检查训练/验证切分后每档样本数是否符合预期

### 模型验证
- 位置分类器验证集准确率作为主指标
- 程度分类器使用：
  - `Top-1` 4 档准确率
  - “相邻档位也算可接受”的宽松准确率
- 至少覆盖：
  - 两个级联
  - 每个级联 4 个程度档
  - 早期、中期、后期样本

### 界面验证
- 上传一张第一级样本，能返回第一级和正确程度区间
- 上传一张第二级样本，能返回第二级和正确程度区间
- 上传非图片文件时给出清晰报错
- 模型文件缺失时给出清晰提示，不静默失败

## Acceptance Criteria
- 对模拟图验证集，级联位置识别应达到可直接使用的稳定水平
- 对模拟图验证集，程度识别允许粗略，但应满足“多数样本落在正确档或相邻档”
- 本地页面可稳定完成单图上传和结果展示
- 预测结果对用户可解释，不只给标签，还给编号范围含义

## Assumptions
- 第一版只面向模拟图，不承诺对真实实验图直接泛化
- 最终线上输入只有“损伤光斑图”，不是“损伤图”
- 程度定义直接绑定模拟序列编号，默认“编号越大，损伤越重”
- 4 档定义固定为等宽区间，不在第一版再做人工重划
- 第一版不兼容目标检测、分割、多目标情况，只做整图分类

### 项目使用手册：

模拟损伤光斑图分级识别基线项目。

当前这套实现面向 `D:\project\img_sorter\模拟图` 下的模拟数据，目标是只根据 `损伤光斑图` 完成两阶段分类：

- 先判断损伤发生在 `第一级` 还是 `第二级`
- 再判断损伤程度落在 `S1` 到 `S4` 的哪个粗略区间
- 提供本地 `Gradio` 页面，支持单张图片上传和结果查看

这套流程与旧的 `app.py` 分开维护，不复用旧业务入口。

## 当前能力

- 自动从本地 `模拟图` 目录构建分类数据集
- 自动生成三套分类训练数据：
  - 级联位置分类器
  - 第一级程度分类器
  - 第二级程度分类器
- 提供训练、评估、单图推理和网页交互入口
- 支持把损伤程度按编号区间映射为 `S1-S4`

## 项目结构

- `damage_classifier/`: 数据构建、预处理、训练、推理、评估、Gradio 页面
- `prepare_damage_dataset.py`: 从本地 `模拟图` 目录生成分类数据集
- `train_damage_models.py`: 训练三套分类模型
- `evaluate_damage_models.py`: 评估两阶段推理链路
- `launch_damage_gradio.py`: 启动本地网页界面
- `README_damage_classifier.md`: 当前这套分类流程的补充说明
- `PLAN_damage_classifier_next_steps.md`: 后续训练和改模型的备忘计划
- `tests/`: 基础单元测试

## 数据规则

模型输入只使用：

- `第一级/损伤光斑图`
- `第二级/损伤光斑图`

当前默认标签规则如下：

- 级联位置标签：直接使用目录名 `第一级` / `第二级`
- 程度标签：根据图片编号按 4 档粗分

默认程度分档为：

- `S1`: `img_001` - `img_030`
- `S2`: `img_031` - `img_060`
- `S3`: `img_061` - `img_090`
- `S4`: `img_091` - `img_121`

## 快速开始

先进入项目并激活虚拟环境：

```powershell
cd D:\project\img_sorter
.\.venv\Scripts\activate
```

### 1. 构建数据集

```powershell
python prepare_damage_dataset.py --clean
```

### 2. 训练模型

CPU 训练示例：

```powershell
python train_damage_models.py --models all --epochs 40 --device cpu
```

如果本机 GPU 可用，可以改成：

```powershell
python train_damage_models.py --models all --epochs 40 --device 0
```

### 3. 评估模型

```powershell
python evaluate_damage_models.py --device cpu
```

### 4. 启动网页界面

```powershell
python launch_damage_gradio.py --device cpu
```

启动后默认访问地址：

```text
http://127.0.0.1:7860
```

## 训练输出位置
