# 模拟损伤光斑图分级识别后续计划

## 1. 这份计划是干什么的
这份文档用于记录当前基线版本已经完成的内容、现阶段训练方式、后续准备怎么继续优化模型，以及后面改动时的注意事项。

目标不是一次性把方案定死，而是给后续迭代留一个清晰的备忘参考。

## 2. 当前已经完成的内容
当前分支已经落地了第一版基线系统，主要包括：

- 基于 `模拟图` 目录自动构建分类数据集
- 两阶段分类流程
  - 模型 A：判断损伤发生在 `第一级` 还是 `第二级`
  - 模型 B：当 A 判断为 `第一级` 时，预测粗略损伤程度 `S1-S4`
  - 模型 C：当 A 判断为 `第二级` 时，预测粗略损伤程度 `S1-S4`
- 固定预处理流程
  - 中心裁切
  - 灰度转 3 通道
  - 缩放到统一输入尺寸
- 本地训练脚本、评估脚本、推理脚本
- 本地 `Gradio` 单图页面

## 3. 当前标签规则
### 3.1 级联位置标签
直接使用目录名作为标签：

- `第一级`
- `第二级`

### 3.2 损伤程度标签
当前先使用文件名编号代表损伤演化顺序，采用 4 档粗分：

- `S1`: `img_001` - `img_030`
- `S2`: `img_031` - `img_060`
- `S3`: `img_061` - `img_090`
- `S4`: `img_091` - `img_121`

这只是第一版的工程定义，后续如果发现和实际物理损伤阶段不够一致，可以重划分档规则。

## 5. 当前基线结果说明
本分支已经完成 1 次冒烟训练和链路验证，目的是证明整套流程能跑通。

当前记录到的基线指标为：

- `cascade_accuracy = 0.50`
- `severity_top1_accuracy = 0.24`
- `severity_relaxed_accuracy = 0.48`

注意：

- 这些结果只是 1 epoch 的 smoke test，不代表最终模型能力
- 当前数值只能说明“代码链路和训练流程是通的”
- 还没有进入正式训练和调参阶段

## 6. 现在最优先要做的事
下一阶段先不要急着改标签体系，也不要先做复杂建模。优先顺序建议如下。

### 6.1 跑正式训练
先在现有标签定义下做一版正式训练，建议：

- `epochs = 40` 起步
- 如果效果仍不稳定，再试 `80`
- 有 GPU 优先用 GPU
- 没有 GPU 先用 CPU 跑通完整结果

推荐命令：

```powershell
python prepare_damage_dataset.py --clean
python train_damage_models.py --models all --epochs 40 --device cpu
python evaluate_damage_models.py --device cpu
python launch_damage_gradio.py --device cpu
```

如果本机 GPU 可用，再把 `cpu` 改成 `0`。

### 6.2 看混淆最严重的地方
正式训练后重点看两件事：

- `第一级` 和 `第二级` 是否能稳定区分
- `S2/S3`、`S3/S4` 这种相邻程度档是否容易混淆

如果多数错误都发生在相邻档之间，说明当前粗分档本身是可接受的，只是边界需要再调整。

### 6.3 用页面实际试图像
训练完成后，不要只看指标，还要用页面实际上传样例图检查：

- 早期损伤图是否经常判成 `S1`
- 中期损伤图是否主要落在 `S2/S3`
- 晚期损伤图是否主要落在 `S4`
- 两级联是否存在明显偏向某一类的现象

## 7. 下一轮可能的优化方向
下面这些是后面准备逐步尝试的方向，不建议一口气全改。

### 7.1 先调训练，再考虑换模型
第一优先不是换大模型，而是先把当前基线调顺：

- 增加训练轮数
- 调整 `imgsz`
- 调整 `crop_size`
- 观察是否过拟合
- 比较 CPU/GPU 完整训练后的差异

### 7.2 优化中心裁切范围
如果当前裁切把有用环纹裁掉，或者保留了太多无信息黑边，会直接影响分类效果。

后面可以重点试：

- 更大中心裁切
- 更小中心裁切
- 不同 `imgsz` 组合

### 7.3 重新审视损伤程度分档
如果 `S1-S4` 的错误主要集中在边界样本，可以考虑：

- 改成不等宽分档
- 让中间阶段更宽一些
- 让早期和后期分档更明确

### 7.4 尝试更合理的评估方式
当前除了 top-1，还用了“相邻档位算可接受”的宽松准确率。后面还可以补：

- 混淆矩阵
- 各档单独准确率
- 各级联分别统计程度识别效果

### 7.5 引入真实图前，先确认模拟图上限
如果模拟图上都分不稳，先不要急着上真实实验图。

更合理的节奏是：

1. 先把模拟图结果做到相对稳定
2. 再评估真实图和模拟图的分布差异
3. 再决定是否需要补标注或做迁移

## 8. 什么时候需要“人工标注”
当前这版任务不需要目标框标注，因为它是整图分类。

只有在以下情况出现时，才建议开始补更细的人工标签：

- 想做黑点位置检测
- 想做损伤区域分割
- 想让“损伤程度”定义更贴近人工判断而不是编号区间
- 想把 `损伤图` 和 `损伤光斑图` 联合建模

换句话说：

- 现在先不用画框
- 先把分类基线模型训起来
- 等第一轮结果出来，再决定是否进入标注阶段

## 9. 后续可能扩展的方向
这部分不是当前必须做，但可以作为后续备选。

### 9.1 让 `损伤图` 参与辅助判断
目前只用 `损伤光斑图` 推理。后面如果发现信息不够，可以尝试：

- 双分支输入
- `损伤图` 辅助判断级联位置
- `损伤图` 辅助判断程度

### 9.2 把程度预测改为回归或排序学习
如果后面希望更贴近“由小到大连续变化”，可以考虑：

- 回归到一个连续分数
- 排序学习
- 分类 + 回归联合输出

### 9.3 区分“模型指标”和“可用性”
后续要注意不要只看准确率数字。真正的使用标准应包含：

- 早中晚三段是否有明显可解释性
- 相邻程度误差是否能接受
- 页面推理结果是否稳定

## 10. 建议的近期执行顺序
建议按下面顺序推进：

1. 先正式训练当前 3 个模型
2. 记录验证指标和页面体验
3. 看主要错在哪一类
4. 调整裁切范围和训练轮数
5. 必要时再调整程度分档
6. 只有在分类思路明显不足时，再考虑补人工标注或换任务形式

## 11. 备忘
后续继续修改时，优先保持下面几件事不乱：

- 保留“先判级联位置，再判程度”的两阶段思路
- 保留独立的训练、评估、推理入口
- 不要把旧的 `app.py` 和这套新流程重新混在一起
- 每次改动后都要重新记录训练参数和验证结果

如果后面计划改动较大，建议直接在这个文档下面继续追加新的阶段记录，而不是把旧结论覆盖掉。
